└──────────────────────────┬───────────────────────────────┘
                           │ HTTPS (token in URL / localStorage)

              ┌────────────────────────────┐
              │   Base44 平台 (低程式碼)     │
              │  · 認證 (auth.me / login)    │

              │  · 整合 (Email/LLM/Upload)   │

### 2.2 重建後架構（自建後端）

```
┌──────────────────────────────────────────────────────────┐
│                     瀏覽器 (React SPA)                      │
│                                                            │
│  App → AuthProvider → QueryClient → Router → Layout        │
│                                                            │
│  Pages ── Components ── Hooks ── lib(業務邏輯)              │
│     │                                                      │
│     ▼                                                      │
│  api/client.js  ← 取代 @base44/sdk 的薄封裝層                │
│  （同樣暴露 entities.X.list/filter/create/update/delete）   │
└──────────────────────────┬───────────────────────────────┘
                           │ REST + JWT (Authorization: Bearer)
                           ▼
              ┌────────────────────────────┐
              │   自建後端 (Node/Python…)    │
              │  · POST /auth/login          │
              │  · GET  /auth/me             │
              │  · CRUD /api/{entity}        │
              │  · 查詢支援 $gte/$lte/$in…    │
              │  · (可選) Email 通知          │
              └─────────────┬──────────────┘
                           ▼
              ┌────────────────────────────┐
              │  資料庫 (PostgreSQL/MySQL…)  │
              │  6 張核心資料表（見第 4 節）  │
              └────────────────────────────┘
```

**重建關鍵**：保持 `api/client.js` 的呼叫介面與現有 `base44.entities.X.*` 一致，
則上層 Pages / Components 幾乎不用改動，只需替換 `src/api/` 與 `src/lib/AuthContext.jsx`。

## 4. 資料模型與依賴（核心）

排休功能依賴 **6 個實體**，關聯如下：

```
        ┌──────────────┐         ┌──────────────┐
        │  Department   │◄────────│   Employee   │──┐ deputy_1 / deputy_2
        │  部門          │ N     M │   員工        │◄─┘ (自我參照，職代)
        └──────────────┘ dept_ids └──────┬───────┘
                                         │ employee_id
                                         ▼
        ┌──────────────┐         ┌──────────────┐
        │  LeaveType    │◄────────│  LeaveRecord │
        │  假別          │ type_id │  請假紀錄     │
        └──────────────┘         └──────────────┘

        ┌──────────────┐         ┌──────────────┐
        │   Holiday     │         │ Notification │ recipient_email
        │  國定假日      │         │  站內通知     │ → Employee.user_emails
        └──────────────┘         └──────────────┘
```

### 4.1 Employee（員工）— 核心

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `name` | string | 中文姓名 |
| `english_name` | string | 英文名（選填）|
| `status` | enum | `active` / `inactive` / `parental_leave` / `hidden` |
| `role` | enum | `admin` / `user`（admin 不需綁定對話框、可看全部）|
| `user_emails` | string[] | 綁定的登入 email（**認證對應鍵**）|
| `department_ids` | string[] | 所屬部門（多對多）|
| `deputy_1` | string | 職代一（Employee id，自我參照）|
| `deputy_2` | string | 職代二（Employee id）|
| `sort_order` | number | 全域排序 |
| `sort_order_by_dept` | object | `{部門id: 排序}` 部門內排序 |

CRUD：`list('name'|'sort_order')`、`filter`、`create`、`bulkCreate`(CSV匯入)、`update`、`delete`

### 4.2 Department（部門）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `name` | string | 部門名（如 DPC、TD-台北）|
| `status` | enum | `active` / `hidden` |
| `sort_order` | number | 顯示順序 |

CRUD：`list('sort_order')`、`create`、`update`、`delete`

### 4.3 LeaveType（假別）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `name` | string | 全名（年假 / 上午休 / 下午休 / 健檢 / 出差…）|
| `short_name` | string | 簡稱（最多 4 字）|
| `color` | string | Hex 顏色 |
| `sort_order` | number | 顯示順序 |

CRUD：`list('sort_order')`、`create`、`update`、`delete`

> **重要陷阱**：請假時段（AM/PM/全天）目前是用 `name` 字串硬判斷
> （`健檢`/`上午休`→AM、`下午休`→PM、其他→全天；`出差`略過警示檢查）。
> 重建時**建議新增 `period` 欄位**到 LeaveType，避免改名就壞掉。見 `lib/leaveUtils.js`。

### 4.4 LeaveRecord（請假紀錄）— 主交易表

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `employee_id` | string | FK → Employee |
| `date` | string | `YYYY-MM-DD`（單日，非區間）|
| `leave_type_id` | string | FK → LeaveType |
| `period` | enum | `full` / `AM` / `PM`（建立時依假別推導後存入）|
| `warning_type` | string[] | `deputy_conflict` / `department_over_limit` |
| `warning_details` | object | 警示細節（衝突者、部門上限數…）|

CRUD：`list`、`filter`、`create`、`bulkCreate`（區間批次）、`update`、`delete`

常用查詢（後端需支援 Mongo 風格運算子）：
```js
// 個人某年區間
filter({ employee_id, date: { $gte: '2026-01-01', $lte: '2026-12-31' } })
// 當日有警示者
filter({ date, warning_type: { $exists: true, $ne: [] } })
```

### 4.5 Holiday（國定假日）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `date` | string | `YYYY-MM-DD` |
| `name` | string | 假日名稱 |
| `type` | enum | `national`（國定）/ `company`（公司特別假）|

CRUD：`list('date')`、`create`、`update`、`delete`
用途：區間排休時跳過週末 + 假日；報表計算工作日。

### 4.6 Notification（站內通知）

| 欄位 | 型別 | 說明 |
|------|------|------|
| `id` | string | 主鍵 |
| `recipient_email` | string | 收件者 email（對應 Employee.user_emails）|
| `type` | enum | `leave_created` / `leave_deleted` |
| `message` | string | 通知文字 |
| `is_read` | boolean | 已讀狀態 |
| `created_date` | datetime | 排序用（`-created_date`）|
| `related_entity_id` | string | 關聯 LeaveRecord id（選填）|
| `related_entity_type` | string | 固定 `LeaveRecord`（選填）|

CRUD：`filter({recipient_email}, '-created_date', 50)`、`create`、`update({is_read})`、`delete`

**收件對象**：建立/刪除請假時，通知 ① 所有 `role==='admin'` 且有綁 email 的員工 ② 該員工的職代（deputy_1/2）。
邏輯集中在 `src/components/utils/leaveNotifications.jsx`，請沿用勿內聯。

## 5. 認證模型

- 取得目前使用者：`base44.auth.me()` → `{ email, role }`
- **員工綁定**：使用者 email 必須出現在某 Employee 的 `user_emails` 陣列中；
  非 admin 且未綁定者，`Layout.jsx` 強制跳綁定對話框。
- 判斷「目前使用者是不是 X」：以 `currentUser.email` 比對 `employee.user_emails`（**不要硬編 email**）。
- 管理功能以 `currentUser.role === 'admin'` 控制。
- 「開發處」身分：動態判斷 boundEmployee 的部門名稱是否含 `開發處`（`src/lib/access.js`，非獨立欄位）。
- Token 來源：`?access_token=` URL 參數 → 寫入 `localStorage('base44_access_token')`。

> 重建後端需提供：`POST /auth/login`、`GET /auth/me`、token 驗證（JWT/Session）。

## 7. 需自建後端取代 Base44 的能力清單

| Base44 SDK 呼叫 | 自建後端對應 |
|-----------------|-------------|
| `auth.me()` | `GET /auth/me` |
| `auth.logout()` / `redirectToLogin()` | `POST /auth/logout` / 登入頁導向 |
| `entities.X.list(sort)` | `GET /api/{entity}?sort=` |
| `entities.X.filter(q, sort, limit)` | `GET /api/{entity}?query=&sort=&limit=` |
| `entities.X.create(data)` | `POST /api/{entity}` |
| `entities.X.bulkCreate([])` | `POST /api/{entity}/bulk` |
| `entities.X.update(id, data)` | `PATCH /api/{entity}/{id}` |
| `entities.X.delete(id)` | `DELETE /api/{entity}/{id}` |
| `appLogs.logUserInApp()` | （可選）`POST /logs` 或直接移除 |
| `integrations.Core.*` | 排休流程未使用，可不實作 |

**查詢語言**：`filter()` 需支援 Mongo 風格運算子 —
`$gte`、`$lte`、`$in`、`$ne`、`$exists`、巢狀欄位等價比對。
這是後端最關鍵的相容點，建議在 `api/client.js` 內把這些運算子轉成後端可解析的 query string 或交給 ORM。

## 9. 領域規則備忘（移植時務必保留）

- **工作日**：`calculateWorkingDays` 排除週六日，但**不含** Holiday，需另外扣。
- **警示門檻**：部門當日請假人數 `>= Math.floor(部門人數 / 3)` 觸發「超過 1/3」警示；`出差` 不計入。
- **批次寫入**：N 筆 update/delete 請走 `runInBatches(items, fn, 10)`，勿用 `Promise.all` 一次轟後端。
- **樂觀更新暫時 id**：用 `crypto.randomUUID()`（需 HTTPS / localhost 安全環境），勿用 `Date.now()`。
- **Radix Select**：禁止 `value=""` 或 `value={null}`，用哨兵字串（如 `"__none__"`）。
- UI 文案一律繁體中文，識別字用英文。
